# Requisitos funcionales del proyecto

# Desarrollar un sistema que permita registrar y gestionar los insumos que se manejan en bodega principal y bodega de sucursales.

# 1. Ingresar con usuario y contraseña. Se manejaran dos usuarios, usuario maestro y usuario de bodega.

# 2. Se divide el rol entre un usuario maestro (control completo del sistema) y usuario de bodega que registra movimientos y genera una solicitud de aprovación. El usuario maestro debe de aprobar o rechazar la solicitud.

# 3. Visualización de insumos ya existentes. Los insumos se mostraran por sucursal (6 sucursales en total) y ademas se dispondra de un listado general que muestre la cantidad de los insumos por sucursal y el total.

# 4. Se podrá editar o eliminar insumos. Solo el usuario maestro podrá editar o eliminar insumos.

# 5. Ingresar nuevos insumos al inventario. Esta debe de ser por bodega, y se debe de registrar a la totalidad del insumo. El usuario de bodega podrá ingresar nuevos insumos al inventario a traves de una solicitud que debera aprovar el usuario maestro. Se le avisará de la solicitud a travez de una alerta o recordatorio en pantalla.

# 6. Registrar historial de entrega de insumos a personal con su nombre, apellido, cantidad, nombre del insumo y fecha. 

# 7. Se puede generar un reporte semanal de los insumos entregados con los detalles de sucursal y personal que lo recibió.
